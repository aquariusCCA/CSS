 頁面布局的整體思路
  - 必須確定頁面的板心 (可視區)，也就是頁面最核心的區域。
  - 分析頁面中的行模塊，以及每個模塊中的列模塊。
  - 一行中的列模塊經常浮動布局，先確定每個列的大小，之後確定列的位置。
  - 製作 HTML 結構，我們還是遵循，先有結構，後有樣式的原則，結構永遠最重要。


確定版心
  - 這個頁面的版心是 1200px，每個版心都要水平居中對齊，可以定義版心為公共類。
  .w {
    width: 1200px;
    margin: auto;
  }


頭部製作
  - 導航欄注意點 : https://www.bilibili.com/video/BV14J4114768?p=201&spm_id_from=pageDriver&vd_source=dd97ccca0358cc54d2813737943d2b54
    * 實際開發中，我們不會直接使用 a 標籤，而是用 li 包含鏈接的做法 ( li + a )。
    * li + a 語意更清晰，一看就是有條理的列表型內容。
    * 如果直接用 a ，搜索引擎容易辨別唯有堆砌關鍵字嫌疑，從而影響網站排名。

    注意 
      -> 讓導航欄一行顯示給 li 加上浮動，因為 li 是塊級元素，需要一行顯示。
      -> 這個 nav 導航欄可以不給寬度，將來可以繼續添加其餘文字。
      -> 因為導航欄裡面文字不一樣多，所以最好給 a 鏈接左右 padding 撐開盒子，而不是添加寬度。

  - search 搜索框 -> 一個 search 大盒子裡面包含 2 個表單控件
    * input 輸入框
    * button 按鈕
